1. Registracija

URI - /account

Za registraciju je potrebno poslati POST request koji mora sadržavati 'AccountId' parametar.

Ukoliko je registracija uspješna, servis će vratiti generirani password za registrirani account.

2. Registracija URL-ova

URI - /register

Ovaj endpoint koristi Basic Authentication, te je u requestu potrebno postavit Authorization header sa username(AccountID) i password(generirani password)

Za registraciju URL-a, potrebno je poslati POST request koji mora sadržavati 'URL' parametar, a može sadržavati i 'redirectType' parametar (Default value je 302)

Ukoliko je registracija uspješna, servis će vratiti generirani (kratki )URL sa {shortKey} koji redirekta na registrirani URL.

Primjer: http://localhost:8080/short/Yzrzb - gdje 'Yzrzb' vrijednost predstavlja {shortKey}

3. Statistika accounta

URI - /statistic/{AccountId}

Ovaj endpoint koristi Basic Authentication, te je u requestu potrebno postavit Authorization header sa username(AccountID) i password(generirani password)

Ukoliko je registracija uspješna, servis će vratiti popis svih registriranih URL-ova koje je korisnik registrirao, te broj posjeta istih

4. Redirekt klijenata

URI - /short/{shortKey}

Get request na ovaj URI vrši redirekt klijenta na prethodno registrirani URL
